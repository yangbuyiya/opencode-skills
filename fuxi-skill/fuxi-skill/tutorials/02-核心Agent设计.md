# 阶段2：核心Agent设计

> Agent是伏羲的"感官系统"，决定了能采集到什么样的信息。

---

## 🎯 目标

设计6个Agent的任务、搜索策略、Prompt模板、输出格式。

完成本阶段后，你将能够：
- 理解6个Agent的设计逻辑和互补关系
- 为每个Agent编写完整的Prompt模板
- 制定信息源策略和优先级
- 处理Agent超时、失败、冲突等异常情况

---

## 📝 关键内容

### 1. 为什么需要6个Agent？

#### 1.1 单一Agent的问题

**场景：让一个Agent"调研Naval的所有信息"**

❌ **问题：**
- 信息过载，不知道重点在哪
- 维度缺失，可能遗漏重要信息
- 缺乏交叉验证
- 结果不可控

#### 1.2 6个Agent的互补关系

| Agent | 维度 | 揭示什么 | 为什么重要？ |
|-------|------|---------|------------|
| 1 著作 | 深度思考 | 系统性思想、核心论点 | 书是深思熟虑的表达 |
| 2 对话 | 即兴反应 | 真实思考过程、应对压力 | 访谈展现未准备的状态 |
| 3 表达 | 语言特征 | 表达风格、高频词汇 | 社交媒体展现日常状态 |
| 4 他者 | 外部视角 | 他人观察、盲点发现 | 自己看不到自己的模式 |
| 5 决策 | 行为模式 | 真实行动、言行一致性 | 行动胜过语言 |
| 6 时间线 | 演变轨迹 | 思想变化、成长路径 | 人是会变的 |

**互补逻辑：**
```
著作（说什么）+ 对话（怎么说）+ 表达（日常风格）
         ↓
    他者（别人怎么看）+ 决策（实际怎么做）
         ↓
    时间线（如何演变）
         ↓
      完整画像
```

### 2. Agent任务分配详解

#### 2.1 Agent 1：著作

**核心任务：** 调研此人的著作和系统性长文。

**为什么是Agent 1？**
- 著作是最结构化的思考
- 信息密度最高
- 可信度最高（本人撰写）

**搜索目标：**
- 书籍（书名、核心论点、出版年份）
- 长篇newsletter/博客/论文
- 反复出现的核心论点（≥3次 = 真信念）
- 自创术语和概念
- 推荐书单（揭示智识谱系）

**提取重点：**
1. **核心论点**：反复出现的观点
2. **自创术语**：独特概念
3. **推荐书单**：智识来源
4. **输入源分析**：受谁影响
5. **智识谱系**：思想流派

**信息源优先级：**
```
作者本人作品 > 官方出版 > 授权转载 > 粉丝整理
```

**验证方法：**
- 多处引用同一论点（≥3次）
- 跨作品一致性检查
- 时间线对比（是否有变化）

**输出文件：** `references/research/01-writings.md`

**输出格式示例：**

```markdown
# [人名] - 著作调研

## 1. 核心著作

### 书名：《[书名]》
- 出版年份：YYYY
- 核心论点：
  1. [论点1] - 来源：第X章 - 可信度：一手
  2. [论点2] - 来源：第Y章 - 可信度：一手

## 2. 反复出现的核心论点（≥3次）

| 论点 | 出现位置 | 频次 | 是否真信念 |
|-----|---------|------|-----------|
| [论点] | 书1、书2、访谈 | 5次 | ✅ |

## 3. 自创术语

| 术语 | 定义 | 来源 |
|-----|------|-----|
| [术语] | [定义] | [出处] |

## 4. 推荐书单

| 书名 | 作者 | 推荐次数 | 说明 |
|-----|------|---------|------|

## 5. 输入源分析

### 影响来源
- 经常引用的作者：[作者列表]
- 思想启蒙者：[人物]
- 概念溯源：[观点]可追溯到[人物]

### 智识谱系
- 思想流派：[流派]
- 跨领域影响：[领域] → [领域]
- 反影响：明确反对[人物]的观点

## 6. 信息来源
- [来源1] - URL - 可信度：一手/二手/推测
- [来源2] - URL - 可信度：一手/二手/推测
```

#### 2.2 Agent 2：对话

**核心任务：** 调研播客、长视频、AMA、深度采访。

**为什么重要？**
- 即兴反应展现真实思考
- 被追问时的回答方式
- 改变立场的瞬间
- 拒绝回答的问题

**搜索目标：**
- 播客访谈（Joe Rogan、Tim Ferriss等）
- 长视频访谈
- AMA（Ask Me Anything）
- 深度采访

**提取重点：**
1. **被追问时的反应**：压力下的思考
2. **即兴类比**：自然的思维跳跃
3. **改变立场的瞬间**：思想灵活性
4. **拒绝回答的问题**：边界和禁忌

**信息源优先级：**
```
官方访谈 > 授权播客 > 公开演讲 > 粉丝转录
```

**验证方法：**
- 交叉验证多个访谈
- 对比不同时期的回答
- 检查是否有矛盾

**输出文件：** `references/research/02-conversations.md`

**输出格式示例：**

```markdown
# [人名] - 对话调研

## 1. 访谈列表

| 访谈名称 | 日期 | 时长 | 平台 | 转录链接 |
|---------|------|------|------|---------|

## 2. 被追问时的回答方式

### 追问：[问题]
- 回答：[原文摘录]
- 来源：[访谈名称] - [时间戳]
- 分析：[如何应对压力]

## 3. 即兴类比

| 类比 | 原文 | 来源 |
|-----|------|-----|
| [类比内容] | [完整句子] | [访谈名称] |

## 4. 改变立场的瞬间

| 原立场 | 新立场 | 改变原因 | 来源 |
|--------|--------|---------|------|

## 5. 拒绝回答的问题

| 问题 | 原因推测 | 来源 |
|-----|---------|------|

## 6. 信息来源
- [来源1] - URL - 可信度
```

#### 2.3 Agent 3：表达

**核心任务：** 调研社交媒体、短文、公开表达。

**为什么重要？**
- 展现日常风格
- 高频用词和句式
- 争议立场和公开辩论

**搜索目标：**
- Twitter/X
- 微博、即刻
- 短文、博客
- 公开辩论

**提取重点：**
1. **高频用词句式**：语言习惯
2. **争议立场**：敢于表态的观点
3. **幽默方式**：个性和风格
4. **公开辩论**：如何应对挑战

**信息源优先级：**
```
官方认证账号 > 本人博客 > 媒体引用 > 粉丝总结
```

**验证方法：**
- 时间线一致性
- 账号归属验证
- 高频词统计

**输出文件：** `references/research/03-expression-dna.md`

**输出格式示例：**

```markdown
# [人名] - 表达DNA

## 1. 高频词汇（Top 20）

| 词汇 | 频次 | 示例句子 |
|-----|------|---------|

## 2. 句式特点

| 句式类型 | 示例 | 频次 |
|---------|------|------|
| "不是...而是..." | [示例] | 15次 |
| "只有...才..." | [示例] | 10次 |

## 3. 修辞手法

- 类比：[示例]
- 隐喻：[示例]
- 自嘲：[示例]
- 讽刺：[示例]

## 4. 情绪基调

- 主基调：[严肃/幽默/犀利/温和]
- 变化：[在不同场景的变化]

## 5. 争议立场

| 立场 | 背景 | 反应 | 来源 |
|-----|------|------|------|

## 6. 公开辩论

| 辩论对象 | 主题 | [人名]立场 | 结果 |
|---------|------|-----------|------|

## 7. 典型句式

- "XXX的秘诀不是Y，而是Z"
- "如果你理解了A，你就会明白B"

## 8. 信息来源
```

#### 2.4 Agent 4：他者

**核心任务：** 调研他人分析、书评、批评、传记。

**为什么重要？**
- 外部视角发现盲点
- 批评揭示争议
- 与同行对比定位

**搜索目标：**
- 书评、评论
- 批评文章
- 他人分析
- 传记、人物报道

**提取重点：**
1. **外部观察到的模式**：别人怎么看
2. **批评与争议**：反对观点
3. **与同行对比**：独特之处

**信息源优先级：**
```
专业书评 > 学者分析 > 媒体报道 > 网友评论
```

**验证方法：**
- 多源交叉验证
- 区分批评者立场
- 识别偏见

**输出文件：** `references/research/04-external-views.md`

---

#### 2.5 Agent 5：决策

**核心任务：** 调研重大决策、转折点、争议行为。

**为什么重要？**
- 行动验证思想
- 言行一致性检查
- 真实价值观体现

**搜索目标：**
- 重大决策（投资、创业、转型）
- 人生转折点
- 争议行为

**提取重点：**
1. **决策背景与逻辑**：为什么这么做
2. **事后反思**：是否后悔
3. **言行一致性**：说的vs做的

**信息源优先级：**
```
本人自述 > 官方公告 > 媒体深度报道 > 他人分析
```

**验证方法：**
- 时间线对照
- 前后言行一致性
- 多方视角对比

**输出文件：** `references/research/05-decisions.md`

---

#### 2.6 Agent 6：时间线

**核心任务：** 调研从出生/出道到现在的完整时间线。

**为什么重要？**
- 思想是演变的
- 识别关键转折点
- 防止使用过时信息

**搜索目标：**
- 出生/出道到现在
- 关键里程碑
- 思想转折点
- **最近12个月动态**（防过时）

**提取重点：**
1. **关键里程碑**：重要事件
2. **思想转折点**：变化时刻
3. **最近动态**：最新信息

**信息源优先级：**
```
官方传记 > 维基百科 > 媒体报道 > 粉丝整理
```

**验证方法：**
- 多源时间线合并
- 事件可查证性
- 标注模糊时间

**输出文件：** `references/research/06-timeline.md`

### 3. Agent Prompt模板设计

#### 3.1 Prompt模板结构

每个Agent的Prompt模板包含：

```
1. 任务说明（你的任务是什么）
2. 搜索方向（去哪里找信息）
3. 输出要求（产出什么）
4. 信息源策略（用什么来源）
5. 质量标准（如何判断）
```

#### 3.2 Agent 1（著作）完整Prompt模板

**真实人物/历史人物模板：**

```
你的任务：调研[人名]的著作和系统性长文。

搜索方向：
- 此人出版的书籍（书名、核心论点、出版年份）
- 长篇newsletter/博客/论文
- 反复出现≥3次的核心论点（这些是真信念）
- 自创术语和概念
- 推荐书单（揭示智识谱系）
- **输入源分析**：
  - 此人经常引用哪些作者/书籍？（影响来源）
  - 此人的书单/推荐阅读清单（智识谱系）
  - 此人公开提到过的思想启蒙者/导师
  - 概念溯源：此人的核心观点可追溯到谁？
- **智识谱系来源**：
  - 思想流派归属（如：奥地利经济学派、斯多葛哲学等）
  - 跨领域影响（如：科学→商业、哲学→投资）
  - 反影响（此人明确反对谁的观点）

输出要求：
- 写入 [skill目录]/references/research/01-writings.md
- 每条信息标注来源URL和可信度
- 区分一手（此人写的）vs 二手（别人总结的）
- 发现矛盾直接记录，不要调和

信息源黑名单：不使用知乎、微信公众号、百度百科。

验证标准：
- 多处引用同一论点（≥3次）才算"真信念"
- 跨作品一致性检查
- 时间线对比（观点是否有变化）
```

**虚拟人物模板变体：**

```
你的任务：调研[角色名]在原著中的表现和设定。

搜索方向：
- 原著文本（小说/剧本/漫画）中的关键场景和台词
- 角色的核心行为模式（在不同场景中的重复行为）
- 角色的价值观冲突和选择（揭示深层性格）
- 作者的创作访谈（角色原型、创作意图）
- 角色≥3次表现出的思维模式（这些是核心设定）
- **角色输入源分析**：
  - 角色在原著中提到过哪些书籍/人物？（角色阅读史）
  - 角色的知识来源（受过什么教育？师从谁？）
  - 角色的思想转变时刻（什么事件/人物改变了角色？）
- **角色智识谱系**：
  - 角色的导师/引路人是谁？
  - 角色所属的学派/组织/传统
  - 角色的思想传承（如果角色是导师型人物，影响了谁？）

输出要求：
- 写入 [skill目录]/references/research/01-writings.md
- 每条信息标注来源（章节/页码/场景）和版本
- 区分"原著设定" vs "改编改动" vs "作者访谈补充"
- 发现不同版本间的设定差异直接记录

信息源策略：允许使用知乎文学分析、公众号影评等深度解读，但需筛选质量。
```

### 4. 信息源策略设计

#### 4.1 信息源优先级表

**真实人物：**

| 来源类型 | 揭示什么 | 权重 | 说明 |
|---------|---------|------|------|
| 本人著作 | 系统性思考 | 最高 | 书籍、论文、长文 |
| 长对话/访谈 | 即兴思维过程 | 最高 | 播客、视频访谈 |
| 实际决策记录 | 真实行为 vs 声称 | 最高 | 投资、创业、转型 |
| 社交媒体 | 表达风格、即时反应 | 中等 | Twitter、博客 |
| 他人评价 | 外部视角、盲点 | 中等 | 书评、传记 |
| 二手转述 | 参考但需验证 | 低 | 媒体报道、粉丝总结 |

**虚拟人物：**

| 来源类型 | 揭示什么 | 权重 | 说明 |
|---------|---------|------|------|
| 原著文本 | 角色所有设定和行为 | 最高 | 小说、剧本、漫画 |
| 作者访谈/创作手记 | 创作意图、角色原型 | 最高 | 作者如何构思 |
| 影视剧本/改编作品 | 视觉化呈现、台词精炼 | 高 | 需标注版本 |
| 文学评论/学术论文 | 深度分析、象征意义 | 中等 | 学者解读 |
| 粉丝深度讨论 | 多角度解读、细节发现 | 中等 | 高质量分析 |
| 同人创作 | 扩展想象、流行解读 | 低 | 需标注"非官方" |

#### 4.2 信息源黑名单

**永远排除（真实人物）：**
- ❌ 知乎：洗稿严重、信息失真率高
- ❌ 微信公众号：封闭生态、无法验证
- ❌ 百度百科/百度知道：信息陈旧且不可靠

**虚拟人物调整：**
- ✅ 知乎：文学、影视类高质量分析可用，需筛选
- ✅ 公众号：影评、书评类深度解读可用，需判断质量
- ❌ 低质量洗稿
- ❌ 与原著矛盾的同人设定
- ❌ 未标注来源的"冷知识"

#### 4.3 中文渠道特殊处理

**接受权威媒体：**
- 36氪、极客公园、晚点LatePost
- 财新、第一财经、虎嗅
- 少数派、机器之心

**接受访谈平台：**
- 播客平台（小宇宙、喜马拉雅原始音频）
- B站原始视频（非搬运号）

### 5. Agent硬性要求

#### 5.1 结果必须写入文件

```
❌ 错误：Agent只返回文本，不存文件
✅ 正确：Agent将结果写入 references/research/0X-xxx.md
```

**为什么？**
- 可追溯：后续可查看原始信息
- 可验证：Phase 2可以引用来源
- 可迭代：质量不够可以补充

#### 5.2 标注来源和可信度

```
每条信息必须标注：
- 来源URL或出处
- 可信度：一手/二手/推测

示例：
"芒格认为逆向思维很重要" - 来源：《穷查理宝典》P.123 - 可信度：一手
```

#### 5.3 区分三种信息类型

```
1. 他说过的：本人原话
2. 别人说他的：他人观察和解读
3. 我推断的：基于信息的推测
```

**示例：**
```
一手：芒格在《穷查理宝典》中写道："反过来想，总是反过来想。"
二手：《滚雪球》中描述芒格经常用逆向思维分析问题。
推测：芒格可能是在投资实践中逐渐形成这个思维习惯。（需标注）
```

#### 5.4 发现矛盾时保留矛盾

```
❌ 错误：调和矛盾，给出"综合版本"
✅ 正确：记录矛盾，说明"存在矛盾"

示例：
2020年访谈：芒格认为比特币是"老鼠药"
2022年访谈：芒格说"比特币有它的价值"

处理：
- 记录两个矛盾观点
- 标注时间和来源
- Phase 2提炼时考虑"思想演变"或"语境差异"
```

### 6. 异常情况处理

#### 6.1 Agent超时

**场景：** 搜索5分钟无有价值结果

**处理：**
```
1. 不等待，继续推进
2. 在调研文件中标注"信息不足"
3. Phase 2在诚实边界中说明
```

#### 6.2 信息源匮乏

**场景：** 总信息条目 < 10条

**处理：**
```
1. Phase 0.5就提醒用户
2. 降低期望：
   - 心智模型减至2-3个
   - 重点提炼表达DNA
3. 增加诚实边界篇幅
```

#### 6.3 Agent结果冲突

**场景：** 不同Agent对同一问题有不同信息

**处理：**
```
1. 保留矛盾
2. 标注来源和时间
3. Phase 2用"内在张力"section收录

示例：
Agent 1著作：观点A
Agent 2对话：观点B（与A矛盾）

处理：
- 两者都记录
- 标注矛盾
- 提炼时考虑"思想演变"或"语境差异"
```

#### 6.4 冷门人物处理

**场景：** 信息太少，无法提炼心智模型

**处理：**
```
方案1：只提炼表达DNA
方案2：明确诚实边界
方案3：建议放弃，推荐其他人物
```

## ❓ 常见问题

### Q1: 6个Agent必须都运行吗？

**A:** 根据人物类型调整：

- **真实在世人物：** 6个全运行
- **历史人物：** 重点1、4、6，其他降级
- **虚拟人物：** 重点1、5、6，其他降级

### Q2: Agent可以并行执行吗？

**A:** 可以，且应该并行：

- 6个Agent相互独立
- 可以同时运行
- 提高效率

### Q3: 信息源太少怎么办？

**A:** 三种处理方式：

1. 降低目标：只提炼表达DNA
2. 明确边界：增加诚实边界
3. 放弃：建议其他人物

### Q4: Agent之间信息矛盾怎么办？

**A:** 保留矛盾，不要调和：

- 记录两个矛盾观点
- 标注时间和来源
- Phase 2考虑"思想演变"或"语境差异"

### Q5: 虚拟人物的信息源策略有什么不同？

**A:** 主要差异：

- 允许使用知乎文学分析（需筛选）
- 允许使用公众号影评（需判断质量）
- 以原著和作者访谈为主
- 同人创作降级为最低权重
