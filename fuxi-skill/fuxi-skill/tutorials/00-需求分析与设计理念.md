## 📝 关键内容

### 1. 问题空间定义

#### 1.1 为什么需要伏羲？

**现有方案的局限性：**

❌ **传统人物档案**：
- 只记录WHAT they said，不提炼HOW they think
- 信息是静态的，无法处理新问题
- 阅读体验像百科全书，没有互动感

❌ **简单提示词**：
- "扮演乔布斯" - 缺乏深度分析
- 容易变成刻板印象的复读机
- 无法处理复杂决策场景

❌ **colleague-skill**：
- 定位是"AI同事"，不是"人物模拟"
- 没有针对真实人物的深度提炼
- 缺少验证机制和质量控制

**伏羲的独特价值：**

✅ **提取思维框架**：
- 不是复制人，是提炼认知模式
- 可处理未见过的场景
- 具有生成力（generative power）

✅ **认知操作系统**：
- 能运行、能迭代、能崩溃
- 有明确的边界和局限
- 用户知道什么时候该信，什么时候不该信

✅ **三重验证机制**：
- 跨域复现、生成力、排他性
- 确保提炼的是"真信念"，不是"随口一说"
- 质量可控，结果可复现

#### 1.2 目标用户是谁？

**核心用户画像：**

👤 **知识工作者**：
- 想学习某个领域专家的思维模式
- 需要在复杂决策前获得"智者视角"
- 希望从多个维度理解问题

👤 **内容创作者**：
- 需要模拟特定人物的写作风格
- 想要深入理解某个IP的表达DNA
- 创作前需要"进入角色"

👤 **研究者/分析师**：
- 需要系统化梳理某人的思想体系
- 对比不同人物的认知差异
- 研究思想演变的轨迹

**使用场景：**

| 场景 | 输入 | 输出 |
|-----|------|------|
| 决策辅助 | "芒格会怎么看这笔投资？" | 芒格视角的分析和决策建议 |
| 写作模拟 | "用纳西姆·塔勒布的风格写一篇关于风险的短文" | 符合塔勒布风格的文章 |
| 思想研究 | "梳理瑞·达利欧的决策框架" | 系统化的决策方法论文档 |
| 角色扮演 | "扮演乔布斯评价这个产品设计" | 乔布斯视角的犀利点评 |

#### 1.3 与colleague-skill的区别

| 维度 | colleague-skill | 伏羲Skill |
|-----|----------------|-----------|
| 定位 | AI同事（通用智能体） | 人物视角模拟（专项智能体） |
| 目标 | 完成任务 | 提炼思维模式 |
| 信息源 | 无预设 | 6路并行采集 |
| 验证机制 | 无 | 三重验证 |
| 输出 | 任务结果 | 认知操作系统 |
| 适用场景 | 日常工作 | 深度思考、决策辅助 |

### 2. 核心价值主张

#### 2.1 提取的是HOW they think，不是WHAT they said

**错误示范：**
```
用户：乔布斯怎么看AI？
错误输出：乔布斯在2010年说过"科技不能替代人文..."（直接引用）
```

**正确示范：**
```
用户：乔布斯怎么看AI？
正确输出：乔布斯会从"人文与科技交汇"的角度思考。
他会问：AI是否让产品变得更简单？是否让用户更有创造力？
他可能不会关心技术细节，但会极度关注AI产品是否"优雅"。
参考：乔布斯反复强调设计的三原则（简单、直观、情感连接）
```

**核心差异：**
- ❌ 复制：记忆、检索、复述
- ✅ 提炼：模式识别、框架抽象、生成推理

#### 2.2 认知操作系统 vs 知识库

**知识库（传统方案）：**
```
输入：问题 → 检索：相关条目 → 输出：信息
特点：静态、无推理能力、无法处理新场景
```

**认知操作系统（伏羲）：**
```
输入：问题 → 运行：心智模型 + 决策启发式 → 输出：推理结果
特点：动态、有生成力、可处理未见场景
```

**类比：**
- 知识库 = 硬盘（存了什么就是什么）
- 认知操作系统 = CPU（能处理新输入）

#### 2.3 可运行的思维框架 vs 静态的人物档案

| 静态档案 | 可运行框架 |
|---------|-----------|
| 列举他说过的话 | 提炼他如何思考 |
| 需要检索才能回答 | 能推理新问题 |
| 遇到档案外问题就卡住 | 有生成力 |
| 不会犯错（因为只是记录） | 有诚实边界（承认局限） |
| 维护成本高（持续更新内容） | 维护成本低（更新框架） |

### 3. 基本概念体系

#### 3.1 心智模型（Mental Models）

**定义：**
一个人看世界的基本镜片，是反复出现的思维模式。

**特征：**
- 跨领域出现（不只是某个专业问题）
- 有解释力（能理解多种现象）
- 有指导力（能指导决策）

**示例（查理·芒格）：**
```
心智模型：逆向思维
描述："反过来想，总是反过来想"
应用场景：投资决策、问题解决、风险识别
提炼依据：在《穷查理宝典》演讲中反复出现（≥15次）
验证方式：
  - 跨域复现：投资、生活、决策都提到
  - 生成力：能推断他对新问题的看法
  - 排他性：不是所有投资者都这样想
```

**如何提炼心智模型？**
1. 在多个场景重复出现（≥3次）
2. 有明确的表述或行为证据
3. 能解释此人的多个决策
4. 能推断此人在新场景的立场

#### 3.2 决策启发式（Decision Heuristics）

**定义：**
做判断时的快速规则，是心智模型在具体场景的应用。

**示例（瑞·达利欧）：**
```
决策启发式：痛苦 + 反思 = 进步
适用场景：面对失败、错误、挫折
应用方式：
  1. 不要逃避痛苦
  2. 记录痛苦的原因
  3. 反思系统如何改进
  4. 建立避免机制
提炼依据：《原则》中反复强调
```

**心智模型 vs 决策启发式：**
- 心智模型：抽象的思维模式
- 决策启发式：具体的行动规则
- 关系：启发式是模型在特定场景的落地

#### 3.3 表达DNA（Expression DNA）

**定义：**
说话的方式和风格，是识别此人的"语音指纹"。

**包含要素：**
1. **高频词汇**：经常使用的词
2. **句式结构**：长短句、排比、反问
3. **修辞手法**：类比、隐喻、自嘲
4. **情绪基调**：严肃、幽默、犀利、温和
5. **话题偏好**：喜欢讨论什么

**示例（纳西姆·塔勒布）：**
```
表达DNA：
- 高频词汇：脆弱、反脆弱、黑天鹅、胖子托尼、火鸡
- 句式特点：喜欢用"不是...而是..."、"只有...才..."
- 修辞手法：讽刺、自嘲、数学类比、古代典故
- 情绪基调：犀利、挑衅、怀疑权威
- 话题偏好：风险、不确定性、概率、历史

典型句式：
"这不是X，这是Y"
"真正的X不是...而是..."
"如果你理解了...，你就会明白..."
```

#### 3.4 反模式（Anti-patterns）

**定义：**
绝对不会做的事，是认知的边界。

**示例（乔布斯）：**
```
反模式：
1. 不会为了功能牺牲简洁
   证据：多次砍掉复杂功能（如iPod初期的许多功能）
   
2. 不会做市场调研驱动的设计
   证据："用户不知道自己想要什么"
   
3. 不会容忍丑陋的设计
   证据：对设计细节的极端追求（即使没人看到）
```

**反模式的价值：**
- 更有辨识度（知道他不会做什么）
- 避免错误预测（不会做出反模式的行为）
- 理解价值观（反模式体现核心原则）

#### 3.5 诚实边界（Honest Boundaries）

**定义：**
Skill承认自己做不到什么，不假装知道。

**类型：**

**时间边界：**
```
"此Skill基于截至2024年1月的信息训练。
对于之后的事件，Skill无法获知。"
```

**信息边界：**
```
"此人在公开场合从未谈论过量子计算。
关于他对量子计算的看法，Skill只能推测。"
```

**能力边界：**
```
"此人没有医学背景。
关于医学专业问题，Skill的建议仅供参考。"
```

**预测边界：**
```
"此人会如何应对未遇到过的极端场景？
Skill无法确定，这是推测。"
```

**为什么诚实边界很重要？**
- 建立信任：用户知道什么时候该信
- 防止滥用：避免在专业领域误导
- 体现尊重：不假装是此人本人

### 4. 人物类型识别

#### 4.1 三种人物类型

| 类型 | 特点 | 信息源 | 提炼难度 | 代表人物 |
|-----|------|--------|---------|---------|
| 真实在世人物 | 信息丰富、有近期动态 | 书、访谈、社交媒体 | 中等 | Naval、马斯克 |
| 历史人物 | 信息固定、无新动态 | 书、传记、档案 | 较高 | 芒格、费曼 |
| 虚拟人物 | 信息有限、需推理 | 原著、改编作品 | 最高 | 福尔摩斯、张无忌 |

#### 4.2 真实在世人物的处理

**优势：**
- 信息源丰富（书、访谈、演讲、社交媒体）
- 有近期动态（可验证是否过时）
- 有他人分析（多视角交叉验证）

**挑战：**
- 信息过载（需要筛选）
- 观点可能变化（需要标注时间）
- 近期动态缺失（需要补充）

**处理策略：**
```python
# Agent 6 时间线必须包含最近12个月动态
if current_date - last_update > 12_months:
    warn("信息可能过时，建议补充近期动态")
```

#### 4.3 历史人物的处理

**优势：**
- 信息固定（不会新增）
- 已有定论（争议较少）
- 时间线完整

**挑战：**
- 缺少对话、社交媒体等维度
- 信息可能散落各处
- 翻译和解读可能有偏差

**处理策略：**
```
1. 调整Agent优先级：
   - 著作（高优先级）
   - 他者分析（高优先级）
   - 对话（低优先级，可能缺失）
   - 表达（低优先级，可能缺失）
   
2. 标注信息来源时间：
   - "1920年的观点"
   - "晚年的反思"
   
3. 使用历史背景补充：
   - "在那个时代..."
   - "考虑到当时的环境..."
```

#### 4.4 虚拟人物的处理

**优势：**
- 创作者已定义完整人设
- 有明确的性格和价值观
- 可以深入分析细节

**挑战：**
- 信息仅限原著（没有真实行为）
- 缺少现实决策案例
- 改编作品可能有偏差

**处理策略：**
```
1. 明确信息源：
   - 原著为主
   - 官方授权作品次之
   - 粉丝创作最低
   
2. 标注"基于虚构"：
   - 诚实边界要明确说明
   - "这是基于小说的推测"
   
3. 从行为推断：
   - 比真实人物更依赖"他做了什么"
   - 而不是"他读了什么书"
```

**虚拟人物的特殊字段：**
```
- 角色输入源：此人读什么书？受谁影响？
- 角色智识谱系：思想流派归属
- 创作者意图：为什么这样设计角色？
```

## ❓ 常见问题

### Q1: 心智模型和决策启发式有什么区别？

**A:** 心智模型是抽象的思维模式，决策启发式是具体的行动规则。

例如：
- 心智模型："逆向思维"（抽象）
- 决策启发式："先想如何失败，再想如何成功"（具体）

### Q2: 为什么要区分三种人物类型？

**A:** 不同类型的信息源不同，处理策略也不同：

- 真实人物：信息丰富，需要验证
- 历史人物：信息固定，需要补充背景
- 虚拟人物：信息有限，需要推理

### Q3: 诚实边界会不会让Skill变得没用？

**A:** 恰恰相反，诚实边界让Skill更有用：

- 建立信任：用户知道什么时候该信
- 防止滥用：避免在专业领域误导
- 体现尊重：不假装是此人本人

### Q4: 如果信息太少，无法提炼心智模型怎么办？

**A:** 有几种处理方式：

1. 降低目标：不追求完整的心智模型，只提炼表达DNA
2. 明确边界：诚实说明信息不足
3. 放弃：有些人物确实不适合做Skill

### Q5: 虚拟人物的Skill有意义吗？

**A:** 有意义，但定位不同：

- 真实人物Skill：决策辅助、思想研究
- 虚拟人物Skill：角色扮演、创作辅助

虚拟人物Skill的价值在于：
- 深度理解IP角色
- 辅助同人创作
- 角色扮演游戏
