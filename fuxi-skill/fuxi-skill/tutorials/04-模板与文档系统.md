### 2. extraction-framework.md设计

#### 2.1 文档的作用

**目标受众：** 使用伏羲Skill创建新Skill的人

**核心内容：** 如何从原始信息提炼心智模型

#### 2.2 extraction-framework完整结构

```markdown
# 伏羲提炼方法论

## 核心原则

### 原则1：提炼HOW they think，不是WHAT they said

**错误示范：**
```
用户：乔布斯怎么看AI？
错误输出：乔布斯在2010年说过"科技不能替代人文..."
```

**正确示范：**
```
用户：乔布斯怎么看AI？
正确输出：乔布斯会从"人文与科技交汇"的角度思考。
他会问：AI是否让产品变得更简单？
```

### 原则2：三重验证确保真信念

一个观点要成为心智模型，必须通过：
1. 跨域复现：在≥2个领域出现
2. 生成力：能推断新问题的立场
3. 排他性：不是所有聪明人都这样想

### 原则3：保留矛盾，不要调和

**错误：** 发现矛盾后，给出"综合版本"
**正确：** 记录矛盾，标注时间和来源

## 提炼流程

### Step 1: 从调研结果中识别候选观点

**来源：**
- Agent调研的6个文件
- 重点关注反复出现的观点

**筛选标准：**
- 出现频次 ≥ 3次
- 在多个场景出现
- 有具体行为支撑

### Step 2: 对候选观点进行三重验证

**验证清单：**
```
[ ] 跨域复现？
    - 在哪些领域出现？
    - 是否跨专业/生活/决策？

[ ] 生成力？
    - 能推断新问题吗？
    - 推理逻辑是否合理？

[ ] 排他性？
    - 是此人独有的吗？
    - 对比其他人的观点
```

### Step 3: 将验证通过的观点组织为心智模型

**组织结构：**
```
心智模型名称：[简洁命名]
定义：[一句话定义]
应用场景：[列出2-3个场景]
提炼依据：[来源和验证]
局限性：[在什么情况下不适用]
```

---

## 提炼优先级

### 高优先级信息

| 信息类型 | 为什么优先 | 来源 |
|---------|-----------|------|
| 反复出现的观点 | 是真信念 | Agent 1/2 |
| 争议立场 | 有辨识度 | Agent 3 |
| 实际决策 | 验证思想 | Agent 5 |

### 低优先级信息

| 信息类型 | 为什么降级 | 处理 |
|---------|-----------|------|
| 单次提及 | 可能偶然 | 需验证 |
| 二手转述 | 可能失真 | 需确认 |
| 矛盾观点 | 需判断 | 保留矛盾 |

## 常见问题

### Q1: 信息太少，无法提炼心智模型怎么办？

**A:** 三种处理方式：
1. 降低目标：只提炼表达DNA
2. 明确边界：增加诚实边界篇幅
3. 建议放弃：推荐其他人物

### Q2: 发现矛盾观点怎么处理？

**A:** 保留矛盾：
- 记录时间和来源
- 分析是否"思想演变"
- 用"内在张力"section收录

### Q3: 如何判断观点是否有生成力？

**A:** 测试方法：
- 选择一个未讨论的问题
- 基于观点推断立场
- 检查推理逻辑是否合理

## 🏗️ 工作原理

### 七层认知结构

伏羲提炼七层认知结构：

| 层级 | 内容 | 说明 |
|-----|------|------|
| 1 | 怎么说话 | 表达DNA（高频词、句式、风格） |
| 2 | 怎么想 | 心智模型（反复出现的思维模式） |
| 3 | 怎么判断 | 决策启发式（具体场景的行动规则） |
| 4 | 什么不做 | 反模式（绝对不会做的事） |
| 5 | 从哪里来 | 智识谱系（影响来源、思想流派） |
| 6 | 在哪里 | 时间线（演变轨迹、最新动态） |
| 7 | 知道局限 | 诚实边界（承认不知道什么） |

### 5个Phase流程

```
Phase 0: 入口分流 → 理解用户意图
    ↓
Phase 1: 多源信息采集 → 6个并行Agent调研
    ↓
Phase 2: 框架提炼 → 三重验证心智模型
    ↓
Phase 3: Skill构建 → 模板填充生成
    ↓
Phase 4: 质量验证 → 3项测试确保质量
```

## 📚 使用场景

### ✅ 适合场景

- **决策辅助：** 获得智者视角的建议
- **写作模拟：** 模拟特定人物的写作风格
- **思想研究：** 系统化梳理某人的思想体系
- **角色扮演：** 深度扮演特定人物

### ❌ 不适合场景

- **事实查询：** Skill不是百科全书
- **专业建议：** 医疗、法律等专业领域请咨询专家
- **预测未来：** Skill无法预测未发生的事件

## ⚠️ 重要说明

### Skill不是真人

- Skill是基于公开信息的思维框架提炼
- 不是此人本人，也无法代表此人
- 遇到未讨论话题会诚实标注"推测"

### 诚实边界

每个Skill都包含诚实边界：
- **时间边界：** 信息有截止日期
- **信息边界：** 某些话题从未讨论
- **能力边界：** 非专业领域的建议仅供参考
- **预测边界：** 无法预测极端场景

## ❓ 常见问题

### Q1: skill-template可以修改吗？

**A:** 可以，但建议保持核心结构：

- 必须包含：心智模型、决策启发式、表达DNA、诚实边界
- 可以添加：特殊section（如"内在张力"）
- 不建议删除：任何核心section

### Q2: extraction-framework是给谁看的？

**A:** 给使用伏羲创建Skill的人：

- 理解提炼方法论
- 学习三重验证
- 掌握质量标准

### Q3: README要多长？

**A:** 平衡完整性和可读性：

- 包含：是什么、怎么用、效果如何
- 长度：500-1500字
- 重点：效果示例和使用场景

### Q4: 为什么用MIT许可证？

**A:** 最宽松，利于传播：

- 允许商业使用
- 允许修改分发
- 只需保留版权声明

### Q5: 文档需要多语言版本吗？

**A:** 看目标用户：

- 主要中文用户：只需中文
- 国际用户：建议中英文双语
- 开源项目：建议英文为主
