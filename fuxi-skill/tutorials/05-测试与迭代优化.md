### 2. 边界情况处理

#### 2.1 信息源匮乏

**场景：** 冷门人物，信息 < 10条

**处理流程：**

```
Step 1: Phase 0.5检测
- 统计信息源数量
- 如果 < 10条，发出警告

Step 2: 提供选项
选项1：降低目标
  - 只提炼表达DNA
  - 不追求完整心智模型

选项2：明确边界
  - 大量诚实边界
  - 标注"信息不足"

选项3：放弃
  - 建议选择其他人物
  - 避免生成低质量Skill
```

**示例：**

```
AI: 检测到信息源不足（仅8条），建议：
    1. 降低目标，只提炼表达DNA
    2. 明确标注信息不足
    3. 选择其他人物
    
用户: 选择1

AI: 将生成简化版Skill，重点提炼表达DNA，心智模型数量减少至2个。
```

#### 2.2 Agent超时或失败

**场景：** 某个Agent搜索5分钟无结果

**处理流程：**

```
Step 1: 超时检测
- Agent运行超过5分钟
- 无有价值结果

Step 2: 不等待
- 标注该Agent"信息不足"
- 继续其他Agent

Step 3: Phase 2处理
- 降低该维度的权重
- 在诚实边界中说明

Step 4: Phase 4验证
- 调整通过标准
- 如不满足最低标准，建议重新调研
```

**示例：**

```
Agent 3（表达）超时：
- 原因：此人不用社交媒体
- 处理：标注"表达维度信息不足"
- 影响：降低风格测试权重
- 诚实边界：说明"此人较少公开表达，表达DNA基于有限样本"
```

#### 2.3 Agent结果冲突

**场景：** 不同Agent对同一问题有矛盾信息

**处理流程：**

```
Step 1: 记录矛盾
- 保留两个矛盾观点
- 标注来源和时间

Step 2: 分析原因
原因1：思想演变
  - 时间不同，观点变化
  - 正常现象

原因2：语境差异
  - 场景不同，表述不同
  - 需要理解上下文

原因3：信息错误
  - 某个来源有误
  - 标注"待验证"

Step 3: 处理方式
- 如果是思想演变：时间线呈现变化
- 如果是语境差异：两个都保留
- 如果是信息错误：标注"矛盾，待验证"
```

**示例：**

```
矛盾案例：
- 2018年：芒格说"比特币是老鼠药"
- 2021年：芒格说"比特币是老鼠药的平方"

分析：
- 不是矛盾，是态度强化
- 时间线呈现：2018 → 2021，态度更坚决

处理：
- 两个观点都保留
- 标注时间
- 提炼心智模型："反对加密货币"
```

#### 2.4 冷门人物处理

**场景：** 真实人物但信息极少

**决策树：**

```
信息源数量？
├─ < 5条 → 建议放弃
├─ 5-10条 → 提供选项（降低目标/明确边界/放弃）
└─ > 10条 → 正常流程，但降低期望
```

**示例：**

```
场景：用户想生成"某个小众艺术家"的Skill

AI: 检测到此人物信息源极少（仅3条公开采访），建议：
    1. 放弃，选择其他人物
    2. 生成极简版（仅表达DNA，心智模型1个）

用户: 选择2

AI: 将生成极简版Skill：
    - 心智模型：1个（基于有限信息）
    - 表达DNA：基于3次访谈
    - 诚实边界：大量标注"信息不足"
    - 通过标准降低
```

### 3. 迭代优化方向

#### 3.1 Agent效率优化

**问题：** Agent搜索效率低，耗时长

**优化方向：**

| 优化项 | 方法 | 预期效果 |
|--------|------|---------|
| 搜索关键词优化 | 基于人物类型调整关键词 | 提高准确率 |
| 信息源预处理 | 建立常用信息源索引 | 减少重复搜索 |
| 并行执行优化 | 优化Agent并行调度 | 降低总耗时 |
| 缓存机制 | 缓存已搜索内容 | 避免重复工作 |

**实施建议：**

```python
# 优化前
for agent in agents:
    agent.run()  # 串行执行

# 优化后
import asyncio

async def run_agents_parallel(agents):
    tasks = [agent.run_async() for agent in agents]
    await asyncio.gather(*tasks)  # 并行执行
```

#### 3.2 Prompt模板优化

**优化示例：**

```
优化前：
```

#### 3.3 验证机制改进

**问题：** 三重验证不够自动化

**优化方向：**

| 优化项 | 方法 |
|--------|------|
| 自动化验证 | AI自动检查跨域复现 |
| 生成力测试 | 自动生成新问题并验证 |
| 排他性对比 | 自动对比其他人物观点 |

**实施建议：**

```python
def validate_mental_model(model, person_name):
    # 验证1：跨域复现
    domains = check_domains(model)
    if len(domains) < 2:
        return False, "未通过跨域复现验证"

    # 验证2：生成力
    test_question = generate_test_question(model)
    inference = infer_stance(model, test_question)
    real_stance = check_real_stance(person_name, test_question)

    if not is_consistent(inference, real_stance):
        return False, "未通过生成力验证"

    # 验证3：排他性
    similar_persons = find_similar_persons(person_name)
    unique = check_uniqueness(model, similar_persons)

    if not unique:
        return False, "未通过排他性验证"

    return True, "验证通过"
```

### 4. 性能优化

#### 4.1 并行执行效率

**当前问题：**
- 6个Agent并行执行，但可能相互干扰
- 总耗时不稳定

**优化方案：**

```python
# 方案1：分组并行
Group 1: Agent 1、2、3（信息采集）
Group 2: Agent 4、5、6（信息采集）

Group 1和Group 2并行执行
Group内部也并行

# 方案2：优先级调度
高优先级Agent先执行（Agent 1、2、5）
低优先级Agent后执行（Agent 3、4、6）

# 方案3：动态调整
根据人物类型动态调整Agent优先级
```

#### 4.2 信息密度评估

**问题：** 如何评估调研质量？

**评估指标：**

| 指标 | 计算方法 | 标准 |
|-----|---------|------|
| 信息条目数 | 统计调研文件条目 | ≥30条 |
| 一手来源占比 | 一手来源/总来源 | >50% |
| 时间覆盖度 | 最近动态占比 | >10% |
| 维度覆盖度 | 6个维度的信息平衡 | 每维度≥5条 |

**质量评分公式：**

```
质量分 = (信息条目数/30) * 0.3
       + (一手来源占比) * 0.3
       + (时间覆盖度/0.1) * 0.2
       + (维度覆盖度) * 0.2
```

#### 4.3 质量评分系统

**设计目标：** 自动评估Skill质量

**评分维度：**

| 维度 | 权重 | 评估内容 |

| 心智模型质量 | 30% | 数量、验证、应用场景 |
| 决策启发式质量 | 20% | 数量、具体性 |
| 表达DNA辨识度 | 20% | 高频词、句式、风格 |
| 诚实边界完整性 | 15% | 四种边界都有 |
| 来源可信度 | 15% | 一手来源占比 |

## ❓ 常见问题

### Q1: 测试需要多少案例？

**A:** 建议：

- 最少：3个案例（真实/历史/虚拟各1个）
- 推荐：6个案例（每类型2个）
- 完整：9个案例（每类型3个）

### Q2: 迭代优化需要多久？

**A:** 取决于质量目标：

- 快速迭代：1-2周
- 深度优化：1-2个月
- 持续改进：长期维护

### Q3: 如何判断优化有效？

**A:** 用质量评分对比：

```
优化前评分：77/100分
优化后评分：85/100分
提升：8分

判断：优化有效
```

### Q4: 边界情况要都处理吗？

**A:** 根据优先级：

- 高频场景：必须处理
- 中频场景：应该处理
- 低频场景：可以处理

### Q5: 性能优化值得投入吗？

**A:** 看使用频率：

- 高频使用：值得深度优化
- 低频使用：基础优化即可
- 平衡：优化投入和收益
