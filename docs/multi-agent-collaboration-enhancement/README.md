# 多Agent协作系统增强方案

## 概述

本文档记录了对伏羲Skill系统提示词版中六路并行调研多Agent系统的增强优化方案。通过引入交叉验证机制、质量评分体系、两轮调研机制等改进，显著提升了调研质量、协作效率和可靠性。

## P0层：核心架构（必须实施）

### P0-1：交叉验证Agent（新增Agent 7）

**目标**：对比六个Agent的调研结果，发现冲突、矛盾、互补点

**实施内容**：
- 新增Agent 7：交叉验证与关系网络
- 在Agent 1-6完成后启动
- 读取所有六个Agent的调研文件进行对比分析

**提取内容**：
- 时间线一致性检查（用Agent 6的成果验证其他Agent）
- 不同来源对同一事件的描述差异
- 标注需要澄清的冲突点
- 人物关系网络（合作者、反对者、影响者）
- 交叉验证发现的可信度调整

**输出文件**：`references/research/07-cross-validation.md`

**协作机制**：
- 检查共享知识库中的关键人物信息
- 识别冲突点后，在Phase 1.6中触发冲突调查
- 与各Agent交叉验证发现的一致性和矛盾
- 发现的关键人物关系更新共享知识库

**实施效果**：
- 能够自动发现信息冲突
- 验证不同维度信息的一致性
- 提供人物关系网络图

---

### P0-2：信息源质量评分系统

**目标**：建立量化的信息源质量评估机制，帮助判断信息可信度

**实施内容**：
每个Agent必须对发现的信息源进行质量评分（1-5分），涵盖四个维度：

#### 评分维度

**1. 权威性（1-5分）**
- 5分：本人著作、官方档案、授权传记
- 4分：权威媒体深访、知名播客完整访谈
- 3分：高质量评论、书评、外部分析
- 2分：一般媒体报道、博客文章
- 1分：社交媒体片段、引用整理

**2. 一手性（1-5分）**
- 5分：本人原话、原文、原始视频/音频
- 4分：第一手观察者记录
- 3分：基于一手材料的分析
- 2分：二手转述
- 1分：多次转述

**3. 完整性（1-5分）**
- 5分：完整内容（全书、完整访谈、全文）
- 4分：主要内容保留
- 3分：重要片段
- 2分：摘要或节选
- 1分：零散引用

**4. 时效性（1-5分）**
- 5分：近6个月内
- 4分：6个月-1年
- 3分：1-3年
- 2分：3-5年
- 1分：5年以上

**评分格式**：
```markdown
**来源**: [书名/文章名/访谈名]
**URL**: [链接]
**质量评分**: 权威性X分 | 一手性X分 | 完整性X分 | 时效性X分 | 综合评分X分
```

**Phase 1.5检查点输出**："可信度热力图"，汇总各Agent的评分分布

**实施效果**：
- 提供量化的信息质量评估
- 帮助识别高质量信息源
- 支持动态任务分配决策

---

### P0-3：两轮调研机制

**目标**：将单次调研改为两轮机制，支持基于质量反馈的迭代优化

**实施内容**：

#### 第一阶段：快速扫描（第一轮）
- 并行启动 6 个子任务/subagent（Agent 1-6）
- Agent 6（时间线）优先启动，建立时间框架
- 其他Agent（1-5）在Agent 6输出后并行启动
- 每个子任务负责一个维度，快速扫描可获得的材料
- 目标：建立初步画像，识别关键信息源和薄弱环节
- 时间限制：每个Agent最多5分钟初步搜索

#### 第二阶段：定向补强（第二轮）
- 基于Phase 1.5检查点结果，识别薄弱维度
- 为信息不足的维度分配更多搜索配额（最多10分钟）
- 针对Agent 7发现的冲突点，启动冲突触发式调查（P1-4）
- 优先补充高质量信息源（综合评分≥4分）

#### 动态任务分配规则
- 如果某维度信息源数量 < 3个，第二轮加倍搜索配额
- 如果某维度一手来源占比 < 30%，第二轮优先搜索一手材料
- 如果Agent 7发现严重冲突，分配专门配额调查冲突点

**实施效果**：
- 支持基于质量反馈的迭代优化
- 动态分配搜索资源
- 提升薄弱维度的信息质量

## P1层：智能优化（强烈推荐）

### P1-4：冲突触发式调查

**目标**：针对Agent 7发现的冲突点，启动专门调查

**实施内容**：
在Phase 1.6（第二轮调研）中实施：
- 为每个冲突点分配3-5分钟搜索配额
- 寻找更多证据支持或解释冲突
- 区分时间演变、语境差异、真实矛盾三种情况
- 在`07-cross-validation.md`中追加冲突调查结果

**实施效果**：
- 主动解决信息冲突
- 提供冲突解释和背景
- 提升最终Skill的一致性

---

### P1-5：搜索路径优化

**目标**：基于时间线和关键节点优化搜索策略

**实施内容**：
- Agent 6（时间线）优先启动，建立时间框架
- 识别关键转折点（重大决策、立场变化、重要合作）
- 在转折点前后6个月增加搜索密度
- 根据Agent 7发现的人物关系，定向搜索相关人物的同期活动
- 近12个月的动态优先级最高

**实施效果**：
- 提升搜索的针对性和效率
- 重点覆盖关键时间节点
- 确保信息的时效性

---

### P1-6：共享知识库

**目标**：建立Agent间信息共享机制，避免重复搜索，提升深度

**实施内容**：

#### 知识库机制
- Agent 6输出后，建立初始共享知识库
- 每个Agent在发现高质量信息源（综合评分≥4分）后，立即更新共享知识库
- 其他Agent在搜索前先检查知识库，避免重复搜索同一来源

#### 跨Agent协作推荐
- Agent 1发现书籍后，Agent 2优先搜索该作者的访谈
- Agent 4发现批评者后，其他Agent可搜索该批评者的同期评价
- 包含引文追踪功能：从一篇高质量材料追踪其引用和被引用

**输出文件**：`references/research/shared-knowledge-base.md`

**知识库格式**：
```markdown
## 共享知识库

### 高质量信息源
- [来源名称] - [Agent发现者] - [综合评分] - [URL]
  - 关键内容摘要
  - 推荐给其他Agent的搜索方向

### 关键人物
- [人物名] - [关系类型] - [发现Agent]
  - 推荐搜索方向
```

**实施效果**：
- 避免重复搜索同一来源
- 提升信息搜索深度
- 实现Agent间智能协作

## P2层：质量保证（建议实施）

### P2-7：覆盖度检查清单

**目标**：为每个维度定义最低覆盖度要求，确保调研完整性

**实施内容**：
每个维度必须满足的最低覆盖度：

#### Agent 1（著作与系统性长文）
- 至少3本核心著作或等量系统性论述
- 至少2个自创术语
- 至少5个重复出现≥3次的观点

#### Agent 2（长对话与深度访谈）
- 至少3个深度访谈（每个≥30分钟）
- 至少2个立场变化时刻
- 至少1个即兴类比

#### Agent 3（表达DNA）
- 至少20个表达样本
- 至少10个高频词
- 至少3个常见句式

#### Agent 4（外部评价与批评）
- 至少3个不同角度的外部评价
- 至少2个常见批评
- 至少1个争议事件

#### Agent 5（重大决策与行为记录）
- 至少3个成功案例
- 至少2个失败案例
- 至少1个言行不一致案例

#### Agent 6（完整时间线）
- 至少10个重要节点
- 至少3个思想转折点
- 近12个月至少3个动态

#### Agent 7（交叉验证与关系网络）
- 至少5个关键人物关系
- 至少2个发现冲突或验证点

**检查机制**：
- 未达标项自动标记为"需要补充"
- 在第二轮调研中重点补充

**实施效果**：
- 确保各维度的最低信息量
- 避免信息严重缺失
- 提供质量量化标准

---

### P2-8：时间分布检查

**目标**：检查信息在时间轴上的分布，避免信息过度集中

**实施内容**：
检查信息在时间轴上的分布：

- 信息分布不能过度集中于任何3年期间（不超过总量的40%）
- 近12个月的动态必须存在，否则标记为"时效性不足"
- 重要转折点前后6个月必须有足够材料支撑
- 早期生涯和近期动态必须都有覆盖

**综合质量评估**：
如果时间分布不均，说明可能存在的认知偏差

**实施效果**：
- 避免信息集中在某一时段
- 确保信息的时效性和连续性
- 识别可能的认知偏差

---

### P2-9：语言和来源多样性检查

**目标**：检查信息源的多样性，确保视野不被局限

**实施内容**：
检查信息源的多样性：

- 至少包含3种不同类型的来源（书籍、访谈、演讲、社交媒体等）
- 如果是国际人物，至少包含2种语言的材料
- 不同平台的来源数量不能过于集中（单一平台不超过50%）
- 一手材料占比应≥30%，否则标记为"一手性不足"

**综合质量评估**：
如果来源多样性不足，说明视野可能受限

**实施效果**：
- 确保信息来源的多样性
- 避免单一平台依赖
- 提升信息的全面性

## Agent协作机制

每个Agent新增"协作"部分，明确与其他Agent的信息共享和交叉验证流程：

### Agent 1（著作）
- 发现高质量书籍后，推荐给Agent 2搜索该作者的访谈
- 关注Agent 6识别的关键时间节点的著作

### Agent 2（访谈）
- 优先搜索Agent 1推荐书籍作者的访谈
- 与Agent 7交叉验证访谈中的时间线一致性

### Agent 3（表达DNA）
- 关注Agent 6识别的关键时间节点的表达样本
- 与Agent 2交叉验证表达方式是否一致

### Agent 4（外部评价）
- 发现批评者后，推荐给其他Agent搜索该批评者的同期评价
- 发现重要争议事件后，推荐给Agent 6在时间线中标注
- 与Agent 7交叉验证外部评价与本人表达的一致性

### Agent 5（决策记录）
- 优先搜索Agent 4识别的争议事件相关决策
- 发现重大决策后，推荐给Agent 6在时间线中标注
- 与Agent 7交叉验证决策与表达的一致性

### Agent 6（时间线）
- 优先启动，建立初始共享知识库
- 输出时间线后，立即推荐关键时间节点给其他Agent
- 识别关键转折点
- 为Agent 7提供时间线验证依据

### Agent 7（交叉验证）
- 在其他六个Agent（1-6）完成后启动
- 读取所有六个Agent的调研文件
- 检查共享知识库中的关键人物信息
- 识别冲突点后，在Phase 1.6中触发冲突调查

## 适用场景

本增强方案适用于以下场景：

1. **人物画像蒸馏**：需要全面了解人物的思维框架和表达方式
2. **主题深度调研**：需要系统性收集和分析某主题的多维度信息
3. **多源信息整合**：需要整合多个来源的信息并发现冲突
4. **质量控制要求高**：需要严格的信息质量控制和验证
5. **复杂决策支持**：需要基于全面可靠的信息进行决策

---

## 后续优化方向

1. **自动化程度提升**：考虑引入更自动化的冲突检测和解决机制
2. **知识库持久化**：将共享知识库持久化存储，支持跨会话复用
3. **评分算法优化**：基于实际使用反馈调整评分维度和权重
4. **Agent专业化**：针对不同类型的人物/主题，调整Agent的专业化方向
5. **用户反馈循环**：建立用户反馈机制，持续优化调研策略
