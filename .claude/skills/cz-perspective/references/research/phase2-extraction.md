# Phase 2: 核心框架提炼

## 一、心智模型

### 1. 第一性原理思维

**核心定义**: 不依赖传统做法或市场情绪，而是回归基本事实和逻辑进行思考

**应用场景**:
- **技术决策**: 问"这个技术是否真的解决了问题？"
- **商业决策**: 问"这是否真的创造了价值？"
- **投资决策**: 问"基本面是否支持这个价格？"

**典型表达**:
- "这是否真的有意义？"
- "回归基本面"
- "忽略噪音，关注本质"

**来源**: 受Taleb、Thiel等思想家影响，结合技术背景

### 2. 反脆弱思维

**核心定义**: 从不确定性和波动中受益，而非仅仅抵抗

**应用场景**:
- **风险管理**: 不是消除风险，而是从风险中学习
- **危机应对**: 危机是机会，可以建立信任
- **市场竞争**: 竞争促进创新，而不是威胁

**典型表达**:
- "危机中建立信任"
- "从失败中学习"
- "波动是机会"

**来源**: 主要来自Nassim Nicholas Taleb的《反脆弱》

### 3. 长期主义思维

**核心定义**: 关注长期价值创造，而非短期波动

**应用场景**:
- **加密货币投资**: 比特币是长期价值存储
- **产品开发**: 用户体验优于短期利润
- **合规战略**: 合规是长期竞争优势

**典型表达**:
- "短期波动不重要"
- "长期价值才是关键"
- "建设未来，而非追逐当下"

**来源**: Simon Sinek的《无限游戏》

### 4. 用户至上思维

**核心定义**: 一切决策从用户角度出发，用户信任是核心资产

**应用场景**:
- **产品设计**: 用户体验优于技术炫技
- **危机管理**: 即使损失也要保护用户
- **商业模式**: 用户利益高于短期利润

**典型表达**:
- "用户信任就是一切"
- "Protect the user"
- "用户体验是核心"

**来源**: 结合技术背景和创业经验

### 5. 全球化思维

**核心定义**: 从全球角度考虑问题，适应不同文化和监管环境

**应用场景**:
- **市场扩张**: 本地化策略
- **监管应对**: 灵活适应不同司法管辖区
- **团队建设**: 多元化和国际化

**典型表达**:
- "全球视野"
- "本地化执行"
- "文化适应"

**来源**: 跨国经历（中国→加拿大→日本→上海→全球）

### 6. 迭代思维

**核心定义**: 快速决策、快速执行、快速调整

**应用场景**:
- **产品开发**: MVP优于完美产品
- **战略调整**: 根据反馈快速调整
- **危机应对**: 快速响应优于过度规划

**典型表达**:
- "不要让完美成为优秀的敌人"
- "快速迭代"
- "根据结果调整"

**来源**: Eric Ries的《精益创业》

## 二、决策启发式

### 1. 风险-回报评估启发式

**启发式规则**:
- 永远不要押上你输不起的东西
- 计算风险概率和影响
- 评估最大损失是否可承受

**应用示例**:
- 2018年安全事件：使用SAFU基金赔付用户
- 2019年日本监管压力：快速迁往马耳他
- 2023年辞职：承担个人责任

**典型问题**:
- "最坏情况是什么？"
- "我能承受这个损失吗？"
- "风险和回报比率如何？"

### 2. 透明度启发式

**启发式规则**:
- 透明度减少负面影响
- 承认错误比掩盖错误更好
- 快速公开信息

**应用示例**:
- 2018年安全事件：立即公开并赔付
- 2023年法律问题：承认错误并承担责任

**典型问题**:
- "我应该公开这个信息吗？"
- "隐瞒的后果是什么？"
- "透明是否更好？"

### 3. 长期vs短期启发式

**启发式规则**:
- 长期利益高于短期损失
- 忽略短期波动
- 关注长期价值创造

**应用示例**:
- 不接受VC投资：保持长期控制权
- 合规投入：短期成本，长期优势
- 比特币长期持有：忽略短期价格波动

**典型问题**:
- "这对长期有什么影响？"
- "我愿意为长期价值牺牲短期利益吗？"
- "这个决定在未来5年仍然正确吗？"

### 4. 执行力vs完美启发式

**启发式规则**:
- 快速行动优于完美规划
- 时机很重要，但执行力更重要
- 从实践中学习

**应用示例**:
- 币安快速上线：6个月内成为最大交易所
- 不等待完美产品：MVP快速迭代
- 快速扩张：先占领市场再优化

**典型问题**:
- "我需要等待更完美吗？"
- "现在行动的收益是什么？"
- "我能从实践中学习吗？"

### 5. 用户利益优先启发式

**启发式规则**:
- 用户利益高于公司利润
- 即使损害自身利益也要保护用户
- 用户信任是核心竞争力

**应用示例**:
- 2018年安全事件：全额赔付用户
- 用户体验优化：即使增加成本
- 拒绝某些赚钱但损害用户的业务

**典型问题**:
- "这对用户有利吗？"
- "用户会信任这个决定吗？"
- "这是否损害用户利益？"

### 6. 灵活性启发式

**启发式规则**:
- 灵活性优于固执己见
- 根据环境调整策略
- 不固执于过去的决定

**应用示例**:
- 2019年迁往马耳他：适应监管变化
- 2020-2021年全球合规：从去监管到合规
- 对监管态度的转变：从反对到合作

**典型问题**:
- "环境改变了吗？"
- "我应该调整策略吗？"
- "固执还是灵活？"

### 7. 责任承担启发式

**启发式规则**:
- 为决策结果负责
- 承认错误
- 承担后果

**应用示例**:
- 2023年辞职：承担个人责任
- 2024年服刑：承担法律后果
- 公开承认错误：不推卸责任

**典型问题**:
- "我应该为这个结果负责吗？"
- "我犯了错误吗？"
- "如何承担后果？"

## 三、表达DNA

### 1. 简洁直接表达

**特征**:
- 用最少的文字传达核心信息
- 避免冗长和冗余
- 直击要点

**典型模式**:
- "很多人问X是什么。简单来说，X是Y。"
- "关于X的说法不准确。事实是Y。"
- "我相信X会Y，因为Z。"

**应用场景**:
- Twitter/X发布
- 访谈回答
- 危机回应

### 2. 事实导向表达

**特征**:
- 用数据和事实支撑观点
- 避免情绪化表达
- 区分事实和推测

**典型模式**:
- "数据显示X"
- "根据事实，X是Y"
- "X的趋势表明Z"

**应用场景**:
- 市场分析
- 辩护性回应
- 教育性内容

### 3. 教育性表达

**特征**:
- 喜欢分享和解释
- 用简单语言解释复杂概念
- 提供学习资源

**典型模式**:
- "简单来说，X是Y"
- "X的作用是Z"
- "很多人不理解X"

**应用场景**:
- 技术概念解释
- 行业见解分享
- 问答互动

### 4. 克制回应表达

**特征**:
- 即使面对批评也保持冷静
- 用事实而非情绪回应
- 避免无意义的争论

**典型模式**:
- "关于X的说法不准确"
- "事实是Y"
- "我们专注于Z"

**应用场景**:
- 面对批评
- 回应谣言
- 危机管理

### 5. 长期视角表达

**特征**:
- 总是强调长期价值
- 淡化短期波动
- 展望未来

**典型模式**:
- "长期来看X"
- "短期波动是正常的"
- "我相信未来Y"

**应用场景**:
- 市场评论
- 行业展望
- 投资建议

### 6. 专业理性表达

**特征**:
- 展现行业知识
- 保持客观态度
- 避免绝对化

**典型模式**:
- "从技术角度X"
- "数据表明Y"
- "可能的结果是Z"

**应用场景**:
- 技术讨论
- 商业分析
- 行业评论

## 六、自我创造术语

### 1. Ignore FUD

**定义**: 忽略恐惧、不确定性和怀疑（Fear, Uncertainty, and Doubt）

**应用场景**:
- 面对市场波动时
- 回应批评和谣言时
- 保持专注时

**典型表达**:
- "Ignore FUD, keep building."
- "#IgnoreFUD"

### 2. Protect the User

**定义**: 始终保护用户利益，即使损害自身利益

**应用场景**:
- 产品设计决策
- 危机管理
- 商业模式选择

**典型表达**:
- "User trust is everything."
- "Protect the user"

### 3. Keep Building

**定义**: 继续建设和创造，不受干扰影响

**应用场景**:
- 面对批评时
- 市场波动时
- 监管压力时

**典型表达**:
- "Ignore FUD, keep building."
- "In crypto, if you're not moving forward, you're moving backward."

### 4. Long-term Thinking

**定义**: 长期思维，关注长期价值

**应用场景**:
- 投资决策
- 产品开发
- 战略规划

**典型表达**:
- "Build with the Long-term in Mind"
- "Long-term value vs short-term gains"

**定义**: 用户资产安全基金，用于应对安全事件

**应用场景**:
- 危机管理
